# Промпт для генерации интерактивной демо-страницы Masker на GitHub Pages

## Контекст продукта

**Masker** — это privacy-first HTTP API для обнаружения и удаления персональных данных (PII) из текста и JSON перед отправкой в LLM (ChatGPT, Claude, Gemini и др.).

### Ключевая ценность продукта

Masker решает критическую проблему: когда разработчики отправляют пользовательские данные в LLM, они рискуют нарушить GDPR/CCPA и подвергнуть пользователей риску утечек. Masker предоставляет простой HTTP API, который автоматически находит и маскирует персональные данные за 15-25ms, без хранения и логирования контента.

### Основные возможности

- **Обработка текста и JSON**: Поддержка как простого текста, так и сложных JSON структур с рекурсивной обработкой
- **4 типа PII**: EMAIL (email адреса), PHONE (международные номера телефонов), CARD (номера карт), PERSON (имена людей через AI/NER)
- **2 языка**: Английский (en) и русский (ru)
- **2 режима маскировки**: 
  - `mask` — заменяет на `***`
  - `placeholder` — заменяет на `<EMAIL>`, `<PERSON>` и т.д.
- **Фильтрация сущностей**: Можно выбрать, какие типы PII маскировать
- **Privacy-first**: Stateless API, без базы данных, без логирования контента, только метаданные
- **Производительность**: Среднее время обработки 15-25ms

### Use cases

- Очистка пользовательских данных перед отправкой в ChatGPT/Claude/Gemini
- Анонимизация тикетов поддержки перед анализом
- Обработка форм с персональными данными
- Санитизация логов перед отправкой в аналитические системы
- Соответствие GDPR/CCPA при использовании сторонних AI сервисов

### Технические детали

- **API endpoint**: `POST /v1/redact`
- **Формат запроса**: JSON с полями `text` или `json`, опционально `language`, `entities`, `mode`
- **Формат ответа**: JSON с `redacted_text`/`redacted_json`, массивом найденных сущностей `items`, и временем обработки `processing_time_ms`
- **Пример запроса**:
  ```json
  {
    "text": "My name is John Doe and my email is john@example.com",
    "mode": "placeholder",
    "entities": ["PERSON", "EMAIL"]
  }
  ```
- **Пример ответа**:
  ```json
  {
    "redacted_text": "My name is <PERSON> and my email is <EMAIL>",
    "items": [
      {"entity_type": "PERSON", "start": 11, "end": 19, "score": 0.85},
      {"entity_type": "EMAIL", "start": 33, "end": 49, "score": 1.0}
    ],
    "processing_time_ms": 15.2
  }
  ```

## Задание для генерации демо-страницы

Создай максимально интерактивную и визуально впечатляющую демо-страницу для продукта **Masker** на GitHub Pages. Страница должна быть статической (HTML/CSS/JS), но выглядеть как премиум SaaS продукт.

### Требования к дизайну

- **Темная тема**: Глубокий темный фон (не просто черный, а градиенты, темно-синий/темно-серый)
- **Премиум визуал**: Современный, дорогой дизайн с акцентом на privacy/security (иконки замков, масок, щитов)
- **Плавные анимации**: Микро-анимации при наведении, появлении элементов, переходах
- **Интерактивность**: Максимально интерактивный интерфейс — пользователь должен видеть результат в реальном времени

### Требования к функциональности

Страница должна включать:

1. **Hero секция** с кратким описанием продукта и CTA
2. **Интерактивный демо-блок**:
   - Поле ввода текста (textarea) с примером текста, содержащего PII
   - Выбор режима маскировки (mask/placeholder)
   - Выбор типов PII для обработки (чекбоксы: EMAIL, PHONE, CARD, PERSON)
   - Выбор языка (en/ru)
   - Кнопка "Redact" для обработки
   - Визуализация результата: исходный текст и обработанный текст рядом (или до/после)
   - Подсветка найденных PII сущностей в исходном тексте
   - Отображение списка найденных сущностей с типами и позициями
   - Показ времени обработки
3. **Секция с примерами use cases** (визуально привлекательные карточки)
4. **Секция с техническими деталями** (API endpoint, примеры запросов/ответов)
5. **Секция "Privacy First"** с акцентом на то, что данные не хранятся

### Технические ограничения

- Статический HTML/CSS/JS (без бэкенда)
- Для демо можно использовать mock API или реальный endpoint `https://masker.kikuai.dev/v1/redact` (если доступен)
- Должна работать полностью офлайн (если используется mock)
- Адаптивный дизайн (mobile-friendly)

### UX требования

- Плавные переходы между состояниями
- Визуальная обратная связь на все действия пользователя
- Интуитивно понятный интерфейс
- Визуализация процесса обработки (loading states)
- Красивое отображение результатов с подсветкой изменений

### Визуальные акценты

- Использовать иконки/символы, связанные с privacy: замки, маски, щиты, глаз с перечеркиванием
- Цветовая схема: темный фон, акцентные цвета для privacy/security (синий, зеленый, возможно фиолетовый)
- Типографика: современные шрифты, хорошая читаемость
- Градиенты и тени для глубины

Фокус на результате: максимально интерактивная, визуально впечатляющая демо-страница, которая показывает ценность продукта Masker.
